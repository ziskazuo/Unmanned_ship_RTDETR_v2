请在当前项目中新增一套“目标级评价指标”，用于海上目标检测任务。我们的任务重点是：

1. 先判断目标有没有被检测到
2. 再判断类别是否正确
3. 不强调预测框和GT框的高精度重合，因此不要再把 IoU-based mAP 作为唯一核心指标

请你实现以下 3 个指标，并提供可运行的评估脚本、清晰注释、以及结果输出：

================================
一、需要实现的指标
================================

1) Detection Rate (Pd)
定义：
对每个 GT 目标，只要存在一个预测命中该 GT，就记为“检测成功”。
公式：
Pd = 检测成功的 GT 数 / 全部 GT 数

2) Classification Accuracy on Detected Targets (CAcc)
定义：
只在“已经检测成功”的 GT 上，统计其匹配预测的类别是否正确。
公式：
CAcc = 检测成功且分类正确的 GT 数 / 检测成功的 GT 数

3) Detection-and-Classification Rate (DCR)
定义：
对每个 GT，只有同时“检测成功”且“分类正确”才算最终成功。
公式：
DCR = 检测成功且分类正确的 GT 数 / 全部 GT 数

请在输出中明确给出这三个指标的数值。

================================
二、命中规则（重点）
================================

我们的任务不强调框回归精度，因此“检测成功”的判断不要使用严格 IoU>=0.5。
请使用“中心点命中”规则：

对每个 GT 和预测框，计算中心点距离：
dist = sqrt((cx_pred - cx_gt)^2 + (cy_pred - cy_gt)^2)

再计算归一化距离：
d_norm = dist / sqrt(w_gt * h_gt)

若 d_norm <= tau，则认为该预测“命中”该 GT。

请把 tau 设为可配置参数，默认值先用：
tau = 0.75

同时支持命令行传参修改 tau。

================================
三、匹配规则（避免重复匹配）
================================

请严格处理一对一匹配，规则如下：

1. 在同一张图内做匹配
2. 一个预测最多匹配一个 GT
3. 一个 GT 最多匹配一个预测
4. 先按预测置信度从高到低排序
5. 对每个预测，寻找所有满足 d_norm <= tau 的未匹配 GT
6. 若有多个候选 GT，优先选择 d_norm 最小的那个
7. 完成匹配后：
   - 该 GT 记为“检测成功”
   - 如果预测类别 == GT 类别，则记为“分类正确”
