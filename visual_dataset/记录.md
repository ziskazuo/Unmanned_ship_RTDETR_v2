loss 是总损失，等于各分支加权求和。你现在这套里主要是：

loss_class：主检测头分类损失（最终 query 的类别对不对）。

loss_bbox：主检测头框回归 L1（中心点和宽高误差）。

loss_piou：主检测头旋转框 IoU 类损失（框几何重叠质量）。

loss_angle：主检测头角度分类损失。

loss_class_aux / loss_bbox_aux / loss_piou_aux / loss_angle_aux：decoder 各辅助层的同类损失，用来稳定训练、加速收敛。

loss_class_dn / loss_bbox_dn / loss_piou_dn / loss_angle_dn：DN（denoising）分支损失，给模型“带噪声的已知目标”监督，提升训练稳定性和收敛速度。

loss_route_primary：route 任务主损失（路线/主航向相关分类监督）。

loss_visible：可见性监督（目标是否可见的二分类）。

loss_cam_class：相机侧 query/候选分类损失。

loss_proj_l1：雷达到相机投影框的 L1 误差。

loss_proj_iou：投影框 IoU 损失。

loss_cam_prompt_class：camera prompt 的类别监督。

loss_cam_prompt_box2d：camera prompt 的 2D 框回归。

loss_cam_prompt_iou2d：camera prompt 的 2D IoU 损失。

loss_cam_prompt_radar_bbox：camera prompt 对应雷达框回归损失。

loss_cam_prompt_radar_iou：camera prompt 对应雷达框 IoU 损失。

loss_cam_prompt_angle：camera prompt 对应角度监督。

loss_cam_dense_score：camera dense score head 的监督损失（你前面加的直接监督核心之一）。

loss_enc_rank：encoder top-k rerank 分支损失（让候选排序更接近 GT 匹配结果）。
